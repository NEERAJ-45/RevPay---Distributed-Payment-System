# Goal Description

The objective is to complete Phase 1 by providing the business logic for the User Service of the Mini UPI simulator. This involves replacing the `UnsupportedOperationException("Not implemented yet")` stubs with actual code for user registration, authentication, JWT token handling, UPI ID generation, and QR Code generation. 

## User Review Required

> [!IMPORTANT]
> The JWT Secret used in the `application.yml` (or environment variables) for `JwtService` must be at least 256 bits (32 characters) long to be secure enough for the HMAC-SHA256 algorithm. I will assume the secret string is sufficient and parse it using `Keys.hmacShaKeyFor(secret.getBytes(StandardCharsets.UTF_8))`. Please ensure the secret configuration meets this length requirement.

### Core Logic & Services

#### [MODIFY] [UpiIdGenerator.java](file:///d:/Neeraj%20Surnis/Prsnl_Project/UPI/upi/user-service/src/main/java/com/neeraj/upi/user/service/UpiIdGenerator.java)
- Implement `generate(String fullName)`:
  - Sanitize the `fullName` to remove spaces and special characters.
  - Use the first word (or full name if one word) in lowercase.
  - Check `userRepository.existsByUpiId()`.
  - If a collision occurs, loop and append a numeric suffix (e.g., `2`, `3`) until a unique UPI ID is found.
  - Append `@miniupi` to the unique prefix.

#### [MODIFY] [JwtService.java](file:///d:/Neeraj%20Surnis/Prsnl_Project/UPI/upi/user-service/src/main/java/com/neeraj/upi/user/service/JwtService.java)
- Implement `generateToken`: Build JWT using `Jwts.builder()`, setting subject, claims (`upiId`, `phone`), issued at, expiration, and signing it with `Keys.hmacShaKeyFor(secret.getBytes())`.
- Implement `validateAndExtract`: Use `Jwts.parser().verifyWith(key).build().parseSignedClaims(token)`.
- Implement extraction methods: Fetch subject (UserId) and claims (UpiId).
- Implement `isTokenValid`: Return a boolean indicating whether `validateAndExtract` succeeds without exceptions.

#### [MODIFY] [QrCodeService.java](file:///d:/Neeraj%20Surnis/Prsnl_Project/UPI/upi/user-service/src/main/java/com/neeraj/upi/user/service/QrCodeService.java)
- Implement `generateQrCodeBase64`: Construct the UPI URI string `upi://pay?pa={upiId}&pn={name}&cu=INR`.
- Use ZXing `QRCodeWriter` to create a `BitMatrix`.
- Write the matrix to a PNG byte array output stream using `MatrixToImageWriter`.
- Return the Base64 encoded string of the byte array.

#### [MODIFY] [UserService.java](file:///d:/Neeraj%20Surnis/Prsnl_Project/UPI/upi/user-service/src/main/java/com/neeraj/upi/user/service/UserService.java)
- Implement `register`:
  - Check if `existsByPhone` to prevent duplicates (throw exception if true).
  - Generate UPI ID using `upiIdGenerator`.
  - Hash the PIN using `passwordEncoder`.
  - Build and save the `User` entity to the repository.
  - Publish the `user.created` event via `eventPublisher`.
  - Generate and return a JWT via `jwtService` in an `AuthResponse`.
- Implement `login`:
  - Fetch user by phone; if not found, throw an unauthorized or bad credentials exception.
  - Verify PIN using `passwordEncoder.matches`; if invalid, throw exception.
  - Generate and return JWT via `jwtService`.
- Implement profile getters (`getByUserId`, `getByUpiId`):
  - Fetch from `UserRepository`, map entity to `UserProfileResponse`, and return. Throw a not found exception if the user does not exist.
